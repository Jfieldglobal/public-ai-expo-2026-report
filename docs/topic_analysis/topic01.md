## 1. Executive Summary

이듬 AI는 사망 후속 행정이라는 극도로 파편화된 공공 서비스 영역을 AI 기반 원스톱 플랫폼으로 전환하는 솔루션을 제시했다. 현재 300개 이상으로 분산된 관련 민원을 단일 여정으로 통합하고, AI Pipeline Orchestrator가 멀티 에이전트 시스템을 통해 접수부터 완결까지 전 과정을 자동 처리한다.

핵심 페르소나는 **김민수(39세, 맞벌이 외동)**다. 부모 사망 후 3대 고충 — ① 전체 로드맵 부재, ② 서류·기관 분산, ③ 기한 준수 리스크 — 을 혼자 감당해야 하는 구조적 현실을 문제의 출발점으로 삼았다. 해결 방향은 자연어 상담 → 맞춤형 민원 목록 생성 → 병렬 API 실행 → 실시간 추적 → 원클릭 제출의 E2E 자동화이며, 데모 앱 **"민원 하나로"**를 통해 약 15분 안에 사망 관련 6개 파생 업무를 완결하는 시나리오를 시연했다.

장기 비전은 사망 신고를 시작으로 출생·결혼·이사·창업 등 **국민 생애주기 전체**를 커버하는 **범정부 표준 플랫폼(Government Case Management Platform)**으로 확장하는 것이다. 목표 가치는 단순 효율화를 넘어 **"배려 행정 실현, 지속 가능한 사회적 가치 창출"**이다.

## 2. Key Technologies

### 2-1. AI Pipeline Orchestrator
전체 민원 처리 워크플로를 조율하는 중앙 제어 엔진. 민원 접수부터 최종 결과 안내까지 8단계 E2E 흐름을 끊김 없이 실행한다.

**E2E 처리 흐름 (8단계)**:
```
국민 접점 → AI 접수 → 분해·계획 → 병렬 실행 → 취합·검증 → 결과 안내 → 피드백
```

### 2-2. 멀티 에이전트 시스템 (Multi-Agent System)
오케스트레이터 아래 4개의 전문화된 서브 에이전트가 역할을 분담한다:

| 에이전트 | 역할 |
|---------|------|
| 의도분류 에이전트 | 민원인의 자연어 입력에서 의도 파악 |
| 민원분해 에이전트 | 복합 민원을 태스크 단위로 분해 |
| 시스템연계 에이전트 | 행정 시스템 API 병렬 호출 |
| 검증·응답 에이전트 | 처리 결과 검증 및 최종 안내 생성 |

### 2-3. 복잡도 기반 에이전트 동적 조합
민원 복잡도에 따라 활성화 에이전트 수를 자동으로 조정한다:

| 처리 경로 | 민원 유형 | 활성 에이전트 수 | 최적화 목표 |
|-----------|-----------|----------------|------------|
| Fast Path | 단순 민원 | 2개 | 속도 |
| Standard Path | 표준 민원 | 4개 | 균형 |
| Full Pipeline | 복합 민원 | 5개 | 정확도 |

### 2-4. RAG (Retrieval-Augmented Generation) 지식 기반
행정 법령, 절차, 서류 요건 등 민원 지식을 실시간 검색·적용하여 정확한 민원 분석과 안내를 제공한다.

### 2-5. MCP (민원 처리 공통 프로토콜) 연계 레이어
기존 레거시 행정 시스템과의 표준화된 연동 인터페이스. 정부24, 국민신문고, 홈택스 등을 단일 프로토콜로 연결한다. (발표 내 LCP = MCP의 오기로 추정)

**연동 대상 시스템**:
- 정부24
- 국민신문고
- 홈택스
- 각 부처·지자체 개별 시스템 (확장형 아키텍처)

### 2-6. 병렬 처리 타임라인 최적화
사망신고 후 파생되는 6개 업무(안심상속 재산조회 → 상속세 신고 등)를 순차·병렬 혼합으로 처리하여 **총 소요 시간 약 15분** 목표를 달성한다.

## 4. Public Sector Impact

### 4-1. 해결하는 문제의 규모
- 사망 관련 연계 민원 **300개 이상** → 단일 플랫폼으로 통합
- 서류 **중복 제출 제거** → 원클릭 제출로 전환
- 기관별 방문 → **비대면 원스톱** 처리

### 4-2. 수혜 계층별 효과

| 수혜자 | 기대 효과 |
|--------|----------|
| 국민 (Citizens) | 비용·시간 절약, 로드맵 불확실성 해소, 기한 누락 방지 |
| 정부 (Government) | 민원 신뢰 제고, 행정 처리 효율화, 인력 절감 |
| 사회 (Society) | 고령사회 맞춤 행정, 1인 가구 지원, 사회적 형평성 강화 |

### 4-3. AS-IS vs TO-BE 비교

| 구분 | AS-IS (현행) | TO-BE (AI 플랫폼) |
|------|-------------|-----------------|
| 연결성 | 자동 연결 부재, 단절·반복 | 끊김 없는 자동 연결 |
| 진행 파악 | 진행 상황 파악 불가 | 실시간 추적 |
| 서류 제출 | 기관마다 반복 제출 | 원클릭 일괄 제출 |
| 소요 시간 | 수일~수주 (다기관 방문) | 약 15분 (병렬 처리) |

### 4-4. 생애주기 확장 로드맵

```
현재 → 사망 후속 행정 (론칭)
확장 → 출생 / 결혼·이혼 / 이사·전입 / 창업·폐업
목표 → 범정부 표준 플랫폼 (Government Case Management Platform)
```

공통 서비스 파이프라인과 **재사용 가능한 컴포넌트** 기반으로 각 생애주기 이벤트로 모듈 확장이 가능하도록 설계됐다.

## 5. Strategic Insights

### 5-1. "배려 행정"이라는 프레임 전략
이듬 AI는 기술 효율화 담론 대신 **사회적 약자 배려**를 핵심 서사로 세웠다. 1인 가구·고령화라는 인구 구조 변화가 만들어낸 행정 공백을 AI로 채운다는 접근은 공공 조달·정책 설득에서 강력한 정당성 근거가 된다.

### 5-2. MCP 표준 선점의 중요성
**MCP(민원 처리 공통 프로토콜)**를 표준 연계 레이어로 정의한 것은 단순 기술 선택이 아니라 **플랫폼 생태계 장악 전략**이다. MCP 표준이 공공 행정에 채택될 경우 이듬 AI는 모든 부처·지자체 연동의 핵심 인프라 공급자 지위를 확보한다.

### 5-3. 동적 에이전트 구성의 운영 경제학
단순/표준/복합 3단계 처리 경로는 속도·비용·정확도를 동시에 최적화하는 구조다. 불필요한 에이전트 활성화 없이 민원 복잡도에 비례하여 자원을 투입함으로써 운영 비용을 통제하면서도 품질을 유지한다.

### 5-4. 범정부 표준 플랫폼 포지셔닝의 리스크
생애주기 전체 커버 전략은 확장성은 크나, **정부24·국민신문고 등 기존 플랫폼과의 역할 충돌** 가능성이 있다. 기존 시스템을 대체가 아닌 연동 대상으로 정의한 점은 현명하나, 범정부 표준으로 채택되려면 행안부·NIA 등 주요 기관과의 협치 구조 설계가 전제되어야 한다.

### 5-5. 현재 성숙도 평가
이듬 AI의 시연은 화면(슬라이드) 기반 데모로 대체되었으며, 실제 행정 시스템과의 실시간 연동은 아직 POC 또는 개념 단계로 보인다. **"민원 하나로"** 앱과 15분 처리 목표는 설득력 있는 비전이나, 안심상속 재산조회·홈택스 API 실연동 여부와 데이터 보안·개인정보 처리 방침은 추가 검증이 필요하다.
