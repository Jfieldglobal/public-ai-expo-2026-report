# Topic 03 — LG CNS Exaone 4.5 모델 적용 및 Workspace 적용 사례 상세 분석

## 1. Executive Summary

LG AI연구원이 개발한 엑사원(Exaone/X1) 4.5는 하이브리드 어텐션·KV 캐시 최적화·멀티 토큰 예측으로 256K 컨텍스트를 처리하고, 19조 토큰 프리트레이닝으로 표현력을 강화한 모델이다. 외부 툴콜링 벤치마크(파워벤치)에서 4.0 대비 23점→70점으로 대폭 향상됐으며, 문서 이해(OCR·차트·다이어그램) 분야에서 대형 모델에 준하는 성능을 소형 모델로 달성했다.

기업 내부 서비스 Chat Exaone은 LG 임직원 8만 명 중 95% 이상이 사용 중이며, 2026년 조달청 디지털서비스몰 PP존 등록으로 정부·지자체 도입 경로를 열었다. 한컴과의 제휴로 연내 한글 공공 문서 에이전트 연동이 예정되어 있어 공공 시장 공략이 가시화됐다.

## 2. Key Technologies

### 2-1. 모델 아키텍처: Exaone 4.5 핵심 기술

**하이브리드 어텐션 (Hybrid Attention)**
- 로컬 윈도우 + 글로벌 윈도우 혼합 구조
- 긴 문맥에서 포지션 혼란으로 인한 답변 붕괴를 완화
- 컨텍스트 길이 256K 처리 실현

**추론 최적화 3종**
- GQA (Grouped Query Attention): 행렬 곱 연산 공유로 중복 계산 제거
- KV 캐시 최적화: 인퍼런스 연산 효율화
- 멀티 토큰 예측: 여러 토큰 동시 생성으로 속도 향상

**프리트레이닝 확장**
- 학습 토큰: 14 trillion → 19 trillion
- 효과: 표현력(capability)·벤치마크 문제 해결력 향상

**다국어 지원 확장**
- 1세대: 한국어·영어 → 스페인어 집중 → 4.5: 독일어·일본어·베트남어 추가
- 6개 주력 언어: 현지인 채용 기반 현지화 최적화 (요약·비교·번역)

### 2-2. 문서 이해 & 에이전트 성능

**문서 이해 (Document Vision)**
- 차트·도표·그래프·표·다이어그램·OCR 처리 특화
- AI2D·Charts 등 벤치마크에서 트릴리언 파라미터급 빅모델에 준하는 성능

**에이전트 루프**
- 지시사항 이해 → 계획 수립 → 수행 → 평가 → 수정 자율 루프
- 외부 툴콜링 파워벤치: 23점 → 약 70점 (4.0 대비)
- 200B+급 모델(KX1)과 유사 성능을 소형 모델로 달성

### 2-3. Exaone PANS (바이오 특화 모델)

- 폐 등 고해상도 조직 이미지에서 암세포·면역세포·항체세포를 세포 단위 분류
- MS/Harvard-Net 선도 분야에서 글로벌 SOTA 1위 달성 (2025년)
- NVIDIA MONAI 플랫폼 등록
- 미국 빅파마·대형 병원과 전략적 제휴 R&D 진행

### 2-4. Chat Exaone: 엔터프라이즈 AI 서비스

**도입 현황**
- 2024년 12월 POC 시작 → LG 임직원 사무직 약 8만 명 중 95%+ 사용

**주요 기능**
- 심층 리서치(Deep Research): 하위 쿼리 분해 → 웹/문서 검색 → 사용자 피드백 → 반복 → 최종 리포트
- 캔버스 모드: 좌측 질의응답 + 우측 최종 산출물 병행·변경 하이라이트 제공
  - 텍스트 캔버스: 문서 완결본 작성
  - 코드 캔버스: 파이썬 샌드박스 실행→에러→수정 반복
- 메모리 기능, 에이전트 CLI, 한글 문서 지원

**성능 비교**
- 문서 업무 최적화 지표에서 GPT-5와 평균 동급
- QN3 235B 대비 4.0 32B가 우수한 성능
- 다문서 기반 QA·프로세스 도식화·추론·요약 전반 우수

### 2-5. Data Foundry: 도메인 데이터 자동 생성·평가

**구조 (창과 방패 모델)**
- 데이터 생성 모델: 현장 문서 업로드 → 질문 생성 → RAG 검색 → 답변·근거 생성 (24시간 GPU)
- 데이터 평가 모델: 중요도·근거 출처·정오 판별 점수화 → 하이퀄리티 데이터 최종 검수

**학습 프로세스 4단계**
1. 프리트레이닝: 대규모 텍스트에서 다음 토큰 예측
2. 포스트트레이닝: 도메인 특화 문서(특허·바이오) 읽기
3. 디스틸레이션 튜닝: 문제집 생성·풀이 기반 (Data Foundry 활용)
4. 강화학습(RLHF): 인간 선호 답변 양식·설명력 강화 (Data Foundry 활용)

**성과**
- 국민연금공단 POC: 튜닝 전 대비 20%+ 성능 개선
- AI 생성 데이터 vs 인간 생성 데이터 튜닝 비교: AI 데이터 10%+ 우위

## 5. Strategic Insights

### 5-1. 소형 모델 전략의 경쟁력

Exaone 4.5의 포지셔닝은 "트릴리언 파라미터 빅모델에 준하는 성능을 소형 모델로"다. 이는 공공 온프레미스 환경에서 GPU 자원 제약 문제를 해결하는 핵심 가치 제안이다. 실제로 QN3 235B 대비 32B 모델이 우수한 결과는 효율 측면의 경쟁 우위를 입증한다.

### 5-2. 엔터프라이즈 검증의 공공 전용 신호

LG 그룹 내 8만 명·95% 사용이라는 수치는 단순 기능 데모를 넘어선 실사용 검증 사례다. 공공 조달에서 "대기업이 이미 대규모로 쓴다"는 레퍼런스는 의사결정 리스크를 줄이는 강력한 근거가 된다.

### 5-3. Data Foundry의 공공 특화 잠재력

공공 도메인은 방대한 법령·행정 문서와 함께 전문가 레이블링 자원이 부족한 환경이다. Data Foundry 방식(AI 자동 생성 → 최종 검수만)이 이 병목을 해소할 수 있으며, 국민연금 20% 개선 사례는 공공 적용 가능성을 구체적으로 뒷받침한다.

### 5-4. 한컴 제휴의 전략적 의미

한컴은 공공 한글 문서 생태계의 사실상 표준이다. Chat Exaone이 한컴 공공 문서 에이전트와 연동되면, 공공 업무에서 가장 마찰이 크던 "한글 서식 작성·법조문 비교" 구간을 직접 공략할 수 있다. 이는 단순 협력을 넘어 공공 시장 Lock-in 전략으로 해석된다.

### 5-5. 리스크

- 256K 컨텍스트·에이전트 성능의 독립 검증 부재 (발표사 측정 벤치마크에 의존)
- 금융(LQAA ETF) 및 바이오(디앤디파마텍) 영역 확장이 핵심 공공 적용과 상이한 도메인
- 한컴 연동·식약처 과제는 연내 완료 목표이나 실제 운영 품질은 추가 확인 필요
