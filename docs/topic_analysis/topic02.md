## 1. Executive Summary

이 세션은 기술 데모가 아니라 **변화관리 이벤트**였다. 행정안전부 황규철 실장(클로드), NIA 김영철 원장(레플릿), 카카오 김세용 부사장(러버블) 등 공공·민간 고위 내빈 3명이 각각 다른 AI 플랫폼으로 "3줄 프롬프트, 5분 이내"에 작동하는 앱을 완성하는 시연을 진행했다.

AI 멘토 신성진 대표의 핵심 메시지는 이중 구조로 설계됐다: **거시 담론(소버린 AI, AX 3대 강국, 반도체)만으로는 조직 내 실행 저항을 돌파할 수 없으니, 개인 PC 단위의 미시 혁신이 그 연결고리**라는 것이다. 시연이 증명한 것은 "제품 완성"이 아니라 "프로토타입 생성 속도의 극단적 단축"이다. 이 차이를 구분하지 못하면 조직은 데모 앱을 운영 서비스로 착각하는 비용을 치르게 된다.

## 2. 시연 구조

### 2-1. 참가자·도구·결과물 매핑

| 시연자 | 소속 | AI 도구 | 결과물 | 상징성 |
|--------|------|---------|--------|--------|
| 황규철 실장 | 행정안전부 | Claude | 지진 재난대응 3D 시뮬레이션 | 공공 안전·재난 분야 비개발자 프로토타이핑 |
| 김영철 원장 | NIA | Replit | AI 정책·예산 현황 대시보드 | 웹 검색 기반 정책 데이터 시각화 |
| 김세용 부사장 | 카카오 | Lovable | 카카오톡형 웹소켓 채팅 서비스 | 풀스택(백엔드 포함) 앱 즉석 생성 |

### 2-2. 각 결과물의 실제 구현 범위

**재난 시뮬레이션 (클로드)**
- 3D 도시 지도 위 지진 진원지·위험 반경·대피 경로 시각화
- 공공 안전 분야 교육·훈련용 프로토타입으로서 가능성 입증
- 실제 재난 대응 시스템 전환 시 요건: 실시간 지리 데이터, 구조물 정보, 센서 연동, 검증된 위험 모델

**AI 정책·예산 대시보드 (레플릿)**
- 웹 검색 도구로 국가 AI 정책·투자 데이터를 수집해 시각화
- 정책 담당자에게 빠른 현황 파악 UI 제공 가능성 시사
- 리스크: 검색 출처 신뢰도, 예산 데이터 최신성, 공식 연동 부재

**카카오톡형 웹채팅 (러버블)**
- 공공 상담 공용방 - 상담원·시민 간 실시간 메시지 교환 구현
- 웹소켓 기반 백엔드까지 포함한 풀스택 생성 주장
- 운영 서비스 요건 미달: 인증·개인정보·메시지 저장·보안 감사 부재

## 3. 핵심 프레임 분석

### 3-1. 거시-미시 이분법

신성진 대표가 설정한 대비 구도:

| 거시 혁신 (이미 있는 것) | 미시 혁신 (이 행사가 제안하는 것) |
|------------------------|--------------------------------|
| 소버린 AI | 개인 PC에서 반복 업무 자동화 |
| AX 3대 강국 도약 | 부서별 작은 문제 즉석 해결 |
| 데이터센터·GPU·반도체 | AI 챔피언이 현장에서 빠르게 결과물 생성 |

이 구도는 거대 정책과 조직 내 실행 저항 사이의 간극을 정확히 짚었다. 공공 조직에서 AI 확산의 병목은 모델 성능이 아니라 "내가 써도 되는가", "공식적으로 인정되는가"라는 심리적 불확실성이기 때문이다.

### 3-2. 세 가지 정치적 효과

1. **권위 부여**: 행안부·NIA·카카오 고위직이 직접 사용 → 개인 취미가 아닌 공식 혁신 도구로 격상
2. **공포 완화**: 비전문가도 프롬프트 몇 줄로 결과가 나온다는 장면 → AI 도구 진입장벽 하락
3. **행동 촉발**: "나도 내 업무에서 시도할 수 있다"는 조직 내 모방 행동 유도

## 5. Strategic Insights

### 5-1. 프로토타입 ≠ 제품: 반드시 구분해야 할 경계

이 시연이 증명한 것을 정확히 표현하면: "AI가 기획 아이디어를 작동 가능한 초안으로 바꾸는 시간을 극단적으로 단축했다." 이것은 강력한 능력이지만, "즉시 제품을 완성했다"와는 다르다.

공공 AX에서 바이브 코딩이 배치되어야 할 위치:
```
아이디어 → [바이브 코딩: 강점 구간] → 프로토타입 
         → [검증·보안·거버넌스: 별도 체계 필요] → 운영 서비스
```

### 5-2. 실제 운영 시 통과해야 할 벽

- 재난 시스템: 지리 데이터·실시간 센서·위험 모델 검증
- 정책 대시보드: 출처 신뢰도·최신성·공식 데이터 연동
- 채팅 서비스: 인증·개인정보보호·메시지 보존·보안 감사
- 공통: 검증되지 않은 코드의 신뢰성, 웹 검색 결과의 공식 데이터 오인 위험

### 5-3. 공공 도입을 위한 추천 접근법

이 행사의 전략적 성과는 **심리적 허가(Psychological Permission)**를 만든 것이다. 다음 단계는 이 에너지를 지속 가능한 AX 체계로 연결하는 것이다:

1. 바이브 코딩 결과물을 공식 실험(PoC)으로 제도화
2. PoC → 운영 전환 기준(보안·데이터·품질) 명시
3. AI 챔피언 네트워크를 통한 사례 공유 구조화
4. 조달·법적 책임 소재 명확화
